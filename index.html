Display and positioning in css

SN Property Description Possible values Outcomes
1 Display Determines the type of render block for an element Block Take up the full width of their container with line breaks before and after, and can have their height and width manually adjusted
Inline Take up as little space as possible, flow horizontally, and cannot have their width or height manually adjusted
Inline-block Can appear next to each other, and can have their width and height manually adjusted.
2 Position It positions HTML elments on a web page Static It is the default position for HTML elements
Relative nables an element to be positioned relative to where it would have originally been on a web page. Accompanying the position declaration with one or more of the following offset properties will move the element away from its default position
Top - moves downwards
Bottom - moves upwards
Left - moves right
Right - moves left
Absolute All other elements will ignore the element and It will be positioned relative to its closest parent element that is positioned with relative or absolute, top ,left. Attributes can be used and if the elments shrinks or expand reset its width or anything to 100%. Since it is removed form the flow when scroled it will not move.
Fixed the box is pinned to a specific sopt on a page. The fixed element stays the same regardless of scrolling. Attributes can be used
sticky keeps an element in the document flow as the user scrolls, but sticks to a specified position as the page is scrolled further. Attribues is used specify from where it should stick
3 Z-index When elements overlap, this property controls how far back or how far forward an element should appear on the web page integer value default value is 0. and Depending on the values, the elements in layered in the browser
4 float Ditermines how far left or how far right an elment shoul float within its parent element and the width should be specified left floats element to the left side
right floats element to the right side
5 clear Specifies how elements should behave when they bump into each other left Left side will not touch any other elements within the same containing element
right Right side will not touch any other elements
both neither side of the element will touch any other element
none element can touch either side
